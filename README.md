# ระบบบริหารสถานะหนี้ดี ครบวงจร (Prototype)

ระบบติดตามและวางแผนยกระดับ **%หนี้ดี (งวด 0-2)** สำหรับ 37 สาขา · 3 เขต · แยกรถใหม่/รถมือสอง

## เปิดใช้งาน
เปิดไฟล์ **`index.html`** ในเบราว์เซอร์ (Chrome/Edge) — เป็นไฟล์เดียวจบ ไม่ต้องติดตั้งอะไร
หากเผยแพร่ผ่าน **GitHub Pages** จะได้ลิงก์เปิดหน้าเว็บได้ทันที (Settings → Pages → Deploy from branch → root)

## 6 ขั้นตอนการทำงาน
1. **โหลดข้อมูล** — นำเข้าสถานะหนี้ 37 สาขา แยกเขต/ประเภท
2. **วางแผน** — ตั้งเป้า%หนี้ดี + วางงานรายงวด (เก็บ0-2/กันไหลตก2→3, ดึงกลับ, ยึด/ฟ้องตัดออก, ขายใหม่, ปิดหนี้ครบกำหนดสัญญา) คำนวณ%หนี้ดีตามแผนสด + สรุปเป้าทุกด้าน
3. **อนุมัติแผน** — ผจก./GM ดูเป้ารวมรายเขต/ภาพรวม แล้วอนุมัติ
4. **อัพเดตผลงาน** — งวด0-2 รายวัน, งวด3-6 (ยึด/ปรับ), งวด7+ (ฟ้อง), ขาย/ปิดบัญชี
5. **ติดตามผล** — ผลจริงเทียบเป้า เลือกดูรายสาขา/เขต/ประเภท/ภาพรวม
6. **ประเมิน/ปิดจบ** — สรุปผลเทียบเป้า %หนี้ดีจริง + %งานสำเร็จ ผ่าน/ไม่ผ่าน

## สิทธิ์ผู้ใช้ (สลับมุมมองมุมขวาบน)
- **สาขา** เห็นเฉพาะตัวเอง+เทียบในเขต · **ซุปการเงิน** เห็นทั้งเขต · **ผจก./GM** เห็นทุกสาขา

## โครงสร้างไฟล์
- `index.html` — ระบบหลักครบวงจร
- `tools/` — เครื่องมือวางแผนรายสาขา + หน้าบันทึกรายวัน (แยกเดี่ยว)
- `docs/` — คู่มือแนวทางการทำงาน, เอกสารสเปกสำหรับทีม IT, เทมเพลต Excel

## หมายเหตุ
ข้อมูลรายวัน/ผลงานในต้นแบบนี้เป็น **การจำลอง** (เป้าอ้างอิงข้อมูลจริง พ.ค. 2569)
ระบบจริงที่หลายคนใช้พร้อมกัน + ล็อกอิน ให้ทีม IT พัฒนาต่อตาม `docs/Debt_System_Spec_for_IT.docx`
